为何是VUI？
----------

在一个尚未稳定的前端团队里面，组员之间的开发技能及理念往往因人而异，这样产生了一些列问题：

（假设A和B负责开发不同的UI组件） 

1. A不喜欢B提供的接口调用方式  

2. A和B使用不同的技巧开发UI组件，前端系统成了一个大杂烩 
  
3. A或者B离职后，其所开发的组件无人能接手  
  
4. 新入职的C难以掌握A或者B当时的开发理念及技巧 
  
5. 产品部门需要“企业级”的产品交付

归根结底在于“三个一致”： 

1. 规格一致（进口车 PK 国产车）  
  
2. 性能一致（jQuery UI官方组件的开发者水平参差不齐）    
  
3. 理念一致（团队自定义需求）



为何选择jQuery UI作为底层？
----------

1. **低学习曲线** 
  - 大部分前端攻城狮都在一定程度上熟悉jQuery UI的开发原理
  - jQuery UI社区资源丰富
 
2. **扩展性**
  - 任意升级jQuery UI底层
